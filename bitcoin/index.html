Decentralized Money

Bitcoin

Bitcoin is an open, decentralized monetary network that allows people to hold and transfer value without relying upon a central bank, payment processor, government database, or private financial intermediary.

Speciedex.org supports Bitcoin as a resilient, borderless, censorship-resistant payment system and as an example of durable public infrastructure built from open-source software, cryptography, distributed consensus, and voluntary participation.

What Bitcoin Is

Bitcoin is a peer-to-peer electronic cash system introduced by Satoshi Nakamoto in 2008 and launched in 2009.

The network records transactions in a public, append-only ledger called the blockchain. Independent computers validate transactions and blocks according to a shared set of consensus rules.

No single company owns Bitcoin. No central server controls the network. Anyone may inspect the protocol, run compatible software, verify the ledger, transmit transactions, or participate in the network.

Why Bitcoin Matters

Bitcoin allows people to verify ownership and transfer value using mathematics, cryptographic signatures, distributed consensus, and independently operated infrastructure.

It can function across borders, outside ordinary banking hours, without requiring a user to obtain permission from a central payment authority.

Bitcoin also demonstrates that globally important infrastructure can be maintained through transparent rules, open-source software, economic incentives, public verification, and decentralized participation.

Protocol Characteristics

Core Properties of Bitcoin

Decentralized

Bitcoin operates through a distributed network of independently controlled nodes, miners, wallets, developers, users, and service providers.

Verifiable

Users may run their own node and independently verify transactions, blocks, monetary issuance, and protocol rules.

Scarce

Bitcoin follows a predetermined issuance schedule enforced by consensus rules and approaches a maximum supply of twenty-one million bitcoin.

Permissionless

Anyone with compatible software and network access may participate without requesting authorization from a central institution.

Open Source

The software and protocol rules can be reviewed, tested, modified, and independently implemented.

Borderless

Bitcoin transactions may be transmitted across geographic and political boundaries through the network.

Censorship Resistant

The distributed design makes it difficult for any single party to prevent valid transactions from being broadcast and confirmed.

Durable

The network is designed to continue operating even when individual servers, companies, jurisdictions, or service providers fail.

Reference Implementation

Bitcoin Core

Bitcoin Core is the most widely used implementation of the Bitcoin protocol and provides the software required to independently validate the network.

Full Validation

A Bitcoin Core full node downloads and validates transactions and blocks according to Bitcoin's consensus rules.

By running a full node, a user does not need to trust a third-party block explorer, wallet provider, exchange, or remote server to determine whether a payment is valid.

Network Participation

Bitcoin Core nodes exchange transactions and blocks with peers, maintain a validated view of the blockchain, and help distribute valid network data.

Publicly reachable nodes may also provide validated blocks and transactions to other participants joining or synchronizing with the network.

Wallet Functions

Bitcoin Core includes wallet functionality for generating addresses, creating transactions, maintaining keys, estimating fees, and managing unspent transaction outputs.

Advanced users may also operate Bitcoin Core as the trusted backend for external wallet interfaces and network services.

Transparent Development

Bitcoin Core is developed publicly through peer review, issue tracking, testing, code review, reproducible releases, and open discussion.

No developer can unilaterally change the rules used by the broader Bitcoin network. Users and node operators decide which software they run.

Network Operation

How the Bitcoin Network Works

Transactions

A Bitcoin transaction transfers control of one or more unspent transaction outputs by satisfying the spending conditions attached to them.

Wallet software creates and signs transactions using cryptographic private keys. The signed transaction is then broadcast to the peer-to-peer network.

Nodes

Full nodes verify that transactions follow Bitcoin's rules. Invalid transactions and invalid blocks are rejected regardless of who created them.

Nodes enforce rules concerning signatures, supply, block structure, transaction validity, and spending history.

Mining

Miners collect valid transactions into candidate blocks and perform proof-of-work in an effort to add the next block to the blockchain.

Proof-of-work makes rewriting confirmed transaction history computationally expensive and provides an objective mechanism for ordering blocks.

Confirmations

A transaction receives its first confirmation when it is included in a valid block.

Additional blocks built afterward increase the cumulative proof-of-work securing that transaction's position in the blockchain.

Issuance and Supply

Bitcoin Monetary Policy

New bitcoin enter circulation through the block subsidy paid to miners who produce valid blocks.

The subsidy is reduced at predetermined intervals through an event commonly called the halving. Over time, the issuance rate declines until the maximum supply approaches twenty-one million bitcoin.

The supply schedule is enforced by full nodes. A block attempting to create more bitcoin than permitted by the consensus rules is rejected as invalid.

This predictable issuance model differs from monetary systems in which supply can be expanded through central policy decisions, emergency programs, or institutional discretion.

Self-Custody

Private Keys and Ownership

Private Keys

A private key is secret cryptographic information used to authorize the spending of bitcoin.

Anyone who obtains a private key may be able to spend the bitcoin controlled by it. Private keys should therefore be protected from theft, loss, accidental disclosure, malware, physical damage, and unauthorized backup access.

Recovery Seeds

Many wallets represent their recovery material as a sequence of words commonly called a seed phrase or recovery phrase.

Anyone with access to the recovery phrase may be able to reconstruct the wallet and control its funds. Recovery material should never be entered into untrusted websites, forms, messages, or software.

Self-Custody

Self-custody means controlling the private keys required to spend bitcoin rather than delegating that control to an exchange or custodian.

Self-custody provides direct control but also places responsibility for backups, security, recovery, and transaction verification upon the user.

Third-Party Custody

Bitcoin held through an exchange or hosted wallet may be controlled by the service provider rather than the account holder.

Custodial users may face counterparty risk, withdrawal limits, account restrictions, insolvency, data exposure, legal seizure, or service interruption.

Protect Recovery Information

Speciedex.org will never request a private key, wallet password, recovery seed, hardware-wallet backup, authentication code, or remote access to a user's wallet.

Transaction Privacy

Bitcoin Is Pseudonymous, Not Automatically Anonymous

Bitcoin addresses do not inherently contain a person's legal name. However, transactions are recorded in a public ledger and may be analyzed.

Address reuse, exchange records, merchant information, network metadata, public posts, wallet behavior, and transaction patterns can connect blockchain activity with real-world identities.

Users should not assume that ordinary Bitcoin transactions are anonymous. Privacy requires careful wallet behavior, address management, network security, operational discipline, and an understanding of blockchain analysis.

Running a full node and using privacy-preserving network tools can reduce dependence upon remote services, but no single tool guarantees anonymity.

Payment Channels

Lightning Network

The Lightning Network is a payment-channel network built on Bitcoin that enables rapid, low-value, and frequent payments without recording every individual transfer directly in the blockchain.

Payment Channels

Two participants may commit bitcoin to a channel and update their balances through cryptographically signed transactions.

The channel can later be settled on the Bitcoin blockchain.

Routed Payments

Lightning payments may travel through multiple connected channels, allowing users to pay recipients with whom they do not share a direct channel.

Routing nodes forward payments according to available liquidity, fees, and network conditions.

LND

LND is an open-source Lightning Network implementation developed by Lightning Labs and used by wallets, developers, node operators, payment services, and infrastructure projects.

LND provides APIs, wallet functions, channel management, routing, invoice handling, and integration with Bitcoin full-node backends.

Operational Responsibility

Running a Lightning node requires attention to liquidity, backups, channel state, connectivity, software updates, security, fees, and recovery procedures.

Lightning can improve payment speed and efficiency, but it introduces additional operational complexity.

Important Considerations

Risks and Responsibilities

Bitcoin transactions are generally irreversible once confirmed. Funds sent to the wrong address, the wrong network, or an incompatible service may not be recoverable.

Bitcoin's exchange value can change significantly over short periods. Holding or using bitcoin involves market, technical, legal, operational, security, custody, and counterparty risks.

Wallet software, exchanges, custodians, mining services, investment products, payment processors, and lending platforms may fail, be compromised, become insolvent, or restrict access to funds.

Users are responsible for understanding the legal, accounting, reporting, and tax requirements that may apply within their jurisdiction.

Nothing on this page is financial, investment, legal, tax, custody, security, or accounting advice.

Security
