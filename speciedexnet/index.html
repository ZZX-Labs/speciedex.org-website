Decentralized Peer-to-Peer Replication and Distribution Network

SpeciedexNet

SpeciedexNet is the networking and replication layer of the Speciedex ecosystem.

It synchronizes the independent blockchain belonging to each permitted species, along with shared indexes, source manifests, evidence objects, protocol releases, and public service metadata.

Nodes may hold selected data or specialized roles, while a full-weight server node preserves a verified one-to-one, one-hundred-percent mirror of the entire corpus.

Every compliant node rejects Homo sapiens records as a permanent protocol rule.

Network Model

A Peer-to-Peer Network of Species Chains

Decentralized Replication

Independent nodes exchange verified blocks, objects, manifests, indexes, releases, and peer information without requiring one permanent central data host.

Distributed Responsibility

No single operator must carry every public function. Full-weight, archival, regional, gateway, media, field, and selective nodes can cooperate while remaining independently operated.

Species-Scoped Synchronization

Nodes may request and verify an individual species chain, selected taxonomic branches, regional packages, recent updates, or the complete corpus.

Complete Mirrors

A full-weight server node stores a one-to-one, one-hundred-percent verified copy of the complete Speciedex network dataset.

Network Participants

Node Roles

Full-Weight Node

Stores and verifies every permitted species blockchain, shared object set, manifests, and required indexes.

Archive Node

Preserves complete historical blocks, releases, source packages, evidence objects, software, and recovery materials.

Regional Node

Prioritizes selected geography, languages, field programs, ecosystems, and locally relevant species.

Selective Node

Follows chosen species, taxa, providers, media classes, or chain windows while verifying received content.

Gateway Node

Provides controlled APIs, feeds, queries, caching, and browser services to clients that do not operate nodes.

Relay Node

Improves peer reachability and propagation without necessarily exposing public search or application services.

Media Node

Stores and serves licensed content-addressed images, audio, video, scans, models, and derivatives.

Field Node

Operates in intermittent or disconnected conditions and synchronizes signed packages when connectivity returns.

Connectivity

Peer Discovery and Transport

Peer Discovery

Nodes discover peers through configured seeds, peer exchange, signed directories, local-network discovery, institutional lists, and manually trusted endpoints.

Multiple Transports

The protocol can operate across ordinary Internet links, Tor services, private institutional networks, mesh links, removable media, delayed transfer, and other documented transports.

Encrypted Sessions

Transport encryption protects sessions in transit while content hashes and signatures independently verify the data itself.

Resumable Exchange

Inventory comparison and chunked object transfer allow interrupted synchronization to continue without restarting a complete dataset.

Data Exchange

Synchronization Protocol

Inventory

Peers compare chain tips, checkpoints, object sets, schema support, releases, and requested subscriptions.

Request

A node requests missing headers, blocks, entries, objects, manifests, indexes, or signed release packages.

Verify

Every received item is checked for identity, ancestry, signatures, schema compatibility, policy compliance, and required dependencies.

Stage

Unverified or incomplete data remains isolated from active indexes and public services.

Commit

Verified updates are atomically committed to local storage and made available to indexing, APIs, feeds, and downstream peers.

Announce

Nodes announce accepted chain tips and available objects without requiring other peers to trust the announcing operator.

Verification

Consensus and Scientific State

Protocol Consensus

Consensus establishes valid ordering, signatures, schemas, policy compliance, chain ancestry, and canonical network state.

Scientific Disagreement

Consensus does not pretend that contested taxonomy or uncertain observations have become scientifically certain. Disagreement is represented as attributable data.

Corrections Without Erasure

Later accepted records can supersede or retract earlier assertions, but the historical sequence remains verifiable.

Human Exclusion

All compliant nodes enforce the permanent rejection of Homo sapiens chains and human-targeted records.

Durability

Resilience and Recovery

Independent Verification

A node can rebuild indexes and verify state from species chains, objects, manifests, and protocol rules.

Snapshots

Signed snapshots accelerate bootstrap while remaining checkable against chain history and published manifests.

Offline Packages

Large or remote deployments can exchange verified releases through removable storage or controlled physical transport.

Multiple Releases

Operators may retain stable, candidate, and historical states for rollback, reproduction, and forensic review.

Fault Isolation

Corrupt peers, invalid blocks, incompatible schemas, and malicious objects remain quarantined and cannot silently replace valid state.

No Single Point of Failure

Independent operators, routes, domains, regions, institutions, and archives reduce dependence on one service.
