Local Node, Validation, Storage, API, and Synchronization Engine

SpeciedexCore

SpeciedexCore is the software foundation used to run the Speciedex protocol from a local system or public server.

It can operate as a complete full-weight node, a selective research node, an institutional mirror, a field station, an API gateway, or an application backend.

A full-weight server node maintains a verified one-to-one, one-hundred-percent copy of the complete Speciedex corpus.

The software permanently rejects Homo sapiens as a species record or tracking target.

Node Software

The Local and Server Node Engine

One Engine, Multiple Systems

SpeciedexCore is designed for desktops, workstations, laptops, servers, institutional systems, embedded field stations, websites, and supported mobile environments.

Local-First Operation

A user can search, inspect, validate, import, export, analyze, and contribute data locally. Network connectivity extends synchronization and submission but does not define ownership of the local dataset.

Full-Weight Mode

A full-weight server node stores and verifies a complete one-to-one copy of every permitted species blockchain and all required shared indexes and manifests.

Selective Mode

Resource-constrained installations may follow selected taxa, regions, sources, media classes, or recent chain windows while preserving cryptographic verification paths.

Capabilities

Core Services

Chain Engine

Creates, validates, stores, indexes, traverses, and repairs the independent blockchain assigned to each permitted species.

Consensus and Policy

Applies protocol rules, validator policy, schema versions, review thresholds, conflict handling, and permanent human-exclusion enforcement.

Provider Ingestion

Imports approved APIs, archives, feeds, institutional exports, field packages, and manually reviewed submissions.

Reconciliation

Links names, records, identifiers, taxa, places, references, media, and observations without erasing source disagreement.

Search and Indexing

Builds full-text, taxonomic, geographic, temporal, identifier, trait, relationship, and provenance indexes.

P2P Synchronization

Discovers peers, exchanges inventories, requests missing blocks and objects, verifies received content, and resumes interrupted transfers.

API and Feed

Exposes read, query, validation, submission, status, streaming, and export interfaces for local and remote clients.

Bitcoin Integration

Supports bounty definitions, invoices, payout records, Lightning settlement, on-chain settlement, and auditable reward state.

Data Architecture

Storage and Verification

Species Chain Store

Maintains block headers, entries, signatures, schemas, ancestry, checkpoints, and canonical state for each species chain.

Object Store

Maintains content-addressed evidence, media metadata, source packages, manifests, maps, and large research objects outside compact chain entries.

Shared Indexes

Maintains cross-species taxonomy, geography, ecological relationship, provider, reference, bounty, and network indexes.

Audit Journal

Records local imports, validation decisions, peer exchanges, administrative actions, failures, repairs, and software migrations.

Cryptographic Integrity

Hashes establish object identity and chain continuity; signatures identify authorized submitters, reviewers, publishers, and protocol releases.

Rebuildability

Indexes and materialized views are treated as reproducible derivatives that can be rebuilt from verified chains, objects, and manifests.

Scientific and Protocol Validation

Validation Pipeline

Schema Validation

Checks required fields, types, units, enumerations, identifiers, encodings, and version compatibility.

Taxonomic Validation

Checks rank, lineage, accepted names, synonyms, source concepts, ambiguous identifications, and proposed new taxa.

Evidence Validation

Checks supporting media, references, specimen records, methods, timestamps, hashes, and evidentiary sufficiency.

Geographic Validation

Checks coordinates, uncertainty, region consistency, habitat plausibility, restricted locations, and impossible movement.

Provenance Validation

Checks source identity, license, acquisition method, transformations, submitter signatures, and duplicate lineage.

Policy Validation

Rejects prohibited human records, privacy-invasive material, malicious payloads, unsupported licenses, and protocol-invalid content.

Conflict Analysis

Detects contradictory assertions and routes them into explicit comparison, review, challenge, or unresolved states.

Commit Validation

Re-verifies ordering, parent linkage, signatures, consensus state, and object availability before accepting a chain update.

Real-Time Feed

Data Input/Output Validation Feed

Ticker-Like Stream

SpeciedexCore exposes a continuous event stream showing new submissions, validation stages, accepted blocks, rejected records, conflicts, reorg-equivalent corrections, peer propagation, bounty state, and chain health.

Machine Interfaces

Clients may consume the feed through documented streaming HTTP, WebSocket, Server-Sent Events, JSON Lines, local sockets, or message-bus adapters.

Human Interfaces

Operators can watch readable status lines, dashboards, timelines, filters, severity levels, species channels, provider channels, and validator explanations.

Deterministic Replay

Feed events include stable identifiers and sequence positions so analyses, dashboards, and incident reviews can be reproduced from a checkpoint.

Trust Boundaries

Security and Constitutional Rules

Permanent Human Exclusion

Homo sapiens is forbidden from being entered as a species blockchain, taxon record, observation target, specimen subject, bounty subject, or derived profile.

The exclusion is constitutional to the protocol and is not an administrative preference that an operator may disable.

Boundary Enforcement

Validation rejects direct human records, aliases, disguised taxonomic entries, human biometric datasets, human genomic profiles, and records whose practical purpose is the tracking or profiling of people.

Incidental references to human activity may appear only as environmental context, provenance, authorship, conservation pressure, or interaction metadata attached to a permitted non-human species record.

Least Authority

Peer access, API keys, signing keys, reviewer roles, wallet controls, publication rights, and administrative services remain separated.

Safe Failure

Invalid updates, corrupt objects, incomplete synchronization, incompatible schemas, and failed migrations remain quarantined and cannot replace verified local state.
